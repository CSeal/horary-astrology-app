# App Icon Design Specification — AstraSk: Horary Chart

**Version:** 1.0
**Date:** 2026-06-04
**Output:** assets/icon.png (iOS + Android), assets/adaptive-icon.png (Android foreground)
**Generator:** scripts/generate-icon.js (Node.js + sharp)

---

## Design Intent

The icon communicates AstraSk's core visual identity in a single glance: a celestial, premium, dark theme with a distinctive gold starburst symbol — the same VerdictStar shape visible throughout the app.

**Design principles:**
- Instantly recognizable at 60×60pt (small list view) AND 1024×1024px (store marketing)
- Premium dark background matches the app's Cosmos Dark theme
- Single clear focal element (starburst) — no text, no complexity
- Differentiates from common astrology apps (Co-Star uses circle/dot, The Pattern uses spiral, Nebula uses gradient sky)

## Color Tokens

Sourced from `src/constants/theme.ts`:

| Role | Hex | Usage |
|---|---|---|
| Background | `#070714` | Icon background fill (matches app's `colors.bgBase`) |
| Starburst fill | `#F5C842` | 8-point star polygon (matches `colors.accentGold`) |
| Glow overlay | `#F5C842` at 0–15% opacity | Radial gradient behind starburst for depth |

---

## Geometry

**Shape:** 8-point starburst polygon (matches VerdictStar SVG component in `src/components/svg/VerdictStar.tsx`)

**Starburst parameters:**
- Center: (512, 512) — center of 1024×1024 canvas
- Outer radius: 307px (30% of canvas width = balanced margin)
- Inner radius: 133px (inner point depth — creates distinctive sharp-star silhouette)
- Points: 8 (octal symmetry — matches the app's star decorative motif)
- Rotation: -90° (first point at 12 o'clock position)

**Radial glow:**
- Type: radialGradient centered at (50%, 50%) r=50%
- Stop 1: `#F5C842` 15% opacity at center
- Stop 2: `#F5C842` 0% opacity at edge
- This creates a soft golden atmosphere without dominating the background

---

## Android Adaptive Icon

The adaptive icon uses the same starburst on a **transparent background** (not the dark background). The dark fill (`#070714`) is provided as the `backgroundColor` in `app.json`'s `android.adaptiveIcon.backgroundColor`.

This ensures the icon renders correctly in:
- Circle crop (Google Pixel launcher)
- Rounded square crop (Samsung, Xiaomi)
- Full bleed crop (some custom launchers)

---

## Generation Script

Run: `npm run generate:icon`

This executes `scripts/generate-icon.js` which:
1. Generates the starburst polygon points using a parameterized function
2. Renders the full icon (dark background + glow + starburst) as SVG
3. Converts to 1024×1024 PNG via `sharp`
4. Outputs `assets/icon.png` (opaque) and `assets/adaptive-icon.png` (transparent background)

**Dependency:** `npm install --save-dev sharp` (already listed in devDependencies after running generate:icon for the first time)

## Manual Design Alternative

If programmatic generation is insufficient (e.g., you want to add custom details in Figma or Illustrator), the design tokens above fully specify the icon. Use these exact colors and the 8-point starburst geometry. Export at 1024×1024 PNG, 72 dpi (pixels matter, not DPI for app icons).

## Quality Checklist (manual review before submission)

- [ ] Icon reads clearly at 60×60pt (thumbnail view)
- [ ] No text in the icon (Apple rejects icons with text that duplicates the app name)
- [ ] No transparency in iOS icon (Apple requires fully opaque PNG)
- [ ] Android adaptive icon: foreground has sufficient padding (safe zone is central 66%)
- [ ] Colors match in-app design (background is #070714, starburst is #F5C842)
- [ ] No other app on the store uses an identical or near-identical icon (verify visually)
