## SDK Upgrade Available: Expo 55 -> 56

Released: 2026-05-21
Target React Native: 0.85.2
Target React: 19.2.3
Migration guide: https://docs.expo.dev/router/migrate/sdk-55-to-56/
Estimated effort: HIGH

### Key breaking changes

1. **expo-router no longer depends on react-navigation** — migration codemod required:
   `npx expo-codemod sdk-56-expo-router-react-navigation-replace [source-dir]`
2. **expo/fetch replaces globalThis.fetch** — opt out via `EXPO_PUBLIC_USE_RN_FETCH=1` if needed
3. **expo-file-system**: `copy()` and `move()` are now async; use `copySync()`/`moveSync()` for sync behavior
4. **@expo/vector-icons no longer bundled** with expo — must add explicitly if used
5. **Minimum Xcode 26.4, iOS/tvOS 16.4, macOS 13.4 required**
6. **TypeScript 6.0.3 required** by SDK 56 (see TypeScript note in Review section)

### Project-specific risks

**nativewind (CRITICAL)** — current version is `5.0.0-preview.4`, a pre-release. Stable npm latest
is 4.2.4 (v4 line). NativeWind v5 is still in preview and has no confirmed SDK 56 compatibility.
Before upgrading SDK, verify nativewind v5 preview works on RN 0.85. If it does not, the entire
styling layer is at risk. This is the single highest-risk item for the SDK upgrade.

**react-native-gesture-handler (HIGH RISK)** — current: 2.30.1, latest: 3.0.0.
Gesture Handler 3 is a major version with significant breaking changes:
  - Legacy architecture support removed (RN 0.82+ required — compatible)
  - New hook-based API (old API still works via gradual migration)
  - `onEnd`/`onFinalize` event object changed: `canceled` property replaces boolean `success` param
  - `PureNativeButton` removed; legacy button components renamed (e.g. `LegacyRectButton`)
  - `GestureDetector` now disrupts view hierarchies; new `InterceptingGestureDetector` / `VirtualGestureDetector` introduced
  - `state` and `oldState` event properties removed
The SDK 56 target may require gesture-handler 3 — confirm in migration guide before upgrading.

**@gorhom/bottom-sheet (HIGH RISK)** — current: 5.x, written for gesture-handler 2 and reanimated 3+.
The library's peer dependency requires `react-native-gesture-handler >=2.16.1`. No confirmed
compatibility with gesture-handler 3 found. If gesture-handler upgrades to 3.0, bottom-sheet
may require a new major version. Verify on the gorhom/react-native-bottom-sheet GitHub before upgrading.

**react-native-reanimated + react-native-worklets pair** — upgrading reanimated to 4.4.0
requires worklets 0.9.x. Current worklets is 0.7.4 and latest stable is 0.9.1. The compatibility
table confirms 4.4.x needs worklets 0.9.x (nightly). Treat these as an atomic update: do not
update one without the other.

**react-native-css** — currently pinned at 3.0.7. Peers require `react-native >=0.81`
and `@expo/metro-config >=54`, both satisfied by SDK 56. No native bridge noted — monitor but
lower risk.

npx expo upgrade
